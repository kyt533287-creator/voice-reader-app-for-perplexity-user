# VoiceReader アプリ UI設計ブリーフ

作成日：2026-02-26
対象：UIデザイン担当者

---

## 1. アプリ概要

**アプリ名：** VoiceReader
**プラットフォーム：** Android
**目的：** テキストを読み上げる（TTS: Text-to-Speech）補助アプリ
**配布予定：** Google Play ストア
**主なユーザー：** 目が疲れやすい人、視覚補助が必要な人、長文を「聞いて」理解したい人

---

## 2. 現在の画面構成（5画面）

```
Main（再生画面）
├── Prompts（プロンプト一覧）
│   └── New/Edit Prompt（プロンプト編集）
└── Dictionary（辞書一覧）
    └── New/Edit Entry（辞書エントリ編集）
```

---

## 3. 各画面の現状

### ① Main（メイン再生画面）

**上部：TopAppBar**
- タイトル：「Voice Reader」
- 右上アイコン：Edit（鉛筆）→ タップするとテキスト編集モード。保存アイコンに切り替わる

**中央：テキスト表示エリア**（Box、白背景・グレーボーダー、高さ固定）
- 読み上げ中の文: 黄色ハイライト
- 停止位置の文: グレーハイライト
- 各文をタップ → その位置から再生開始
- 編集モード時は OutlinedTextField に切り替わる

**中下部：進行バー（条件付き表示）**
- テキストがスクロールできる長さのときのみ表示
- 読み上げ進行度を % で表示（アニメーション付き）

**下部コントロールエリア：**

| 行 | ボタン |
|----|--------|
| 1段目（スライダー） | Speed スライダー / Pitch スライダー |
| 2段目（アクション） | Play/Stop ・ PDF/Word/GDoc ・ Paste |
| 3段目（ナビ） | Prompts ・ Dict ・ Aa（フォントサイズ） |

- **Play/Stop**：再生中は赤（Stop）、停止中は青（Play）
- **PDF/Word/GDoc**：ファイルピッカーを開く（PDF・Word・Google Doc対応）
- **Paste**：クリップボードのテキストを貼り付け
- **Prompts**：プロンプト管理画面へ遷移（tertiary色）
- **Dict**：辞書管理画面へ遷移（secondary色）
- **Aa**：フォントサイズを S(13sp) → M(16sp) → L(20sp) と循環切替

**特記事項：**
- 編集モード中にバックジェスチャーを行うと「Leave without saving?」ダイアログが出る（NO=青塗りつぶし・目立つ、YES=アウトライン・控えめ）

---

### ② Prompts（プロンプト一覧）

- TopAppBar：タイトル「Prompts」・戻るボタン・＋ボタン
- プロンプトのカード一覧（タイトル・内容プレビュー）
- カードをタップ → 内容をクリップボードにコピーしてメイン画面に戻る
- 編集・削除アイコン（Edit=青、Delete=赤）

---

### ③ New/Edit Prompt（プロンプト編集）

- Title フィールド（1行）
- Content フィールド（複数行、全画面を使う）
- Save ボタン（大きい・幅いっぱい）
- 変更がある状態でバック → 「Leave without saving?」ダイアログ

---

### ④ Dictionary（辞書一覧）

- 読み上げたくない単語や読み替え単語の登録リスト
- カードに「元の単語 → 置換後（または skip）」表示
- ON/OFF スイッチ・Edit・Delete
- 空状態にはサンプルと使い方のガイドテキスト

---

### ⑤ New/Edit Entry（辞書エントリ編集）

- 「Word to replace」フィールド（置き換え前の単語）
- 「Replacement」フィールド（空欄なら skip 扱い）
- Enable this entry スイッチ
- Save ボタン（大きい・幅いっぱい）
- 変更がある状態でバック → 「Leave without saving?」ダイアログ

---

## 4. テキスト入力方法（Main画面）

以下の4つの方法でテキストを読み込める：

| 方法 | 操作 |
|------|------|
| クリップボード貼り付け | Paste ボタン |
| PDF | PDF/Word/GDoc ボタン → ファイル選択 |
| Wordファイル（.docx） | PDF/Word/GDoc ボタン → ファイル選択 |
| Google Doc | PDF/Word/GDoc → Doc選択 → Google DocsアプリでDoc が開く → コピー → Paste |
| URLシェア | 他アプリから「共有」でURLを送る → 本文が自動取得される |

## 5. 未着手・改善余地のある領域（UI設計のヒント）

- **全体的なビジュアルデザイン**：現在はMaterial3のデフォルトカラー。ブランドカラーや統一感のある配色への変更が期待される
- **テキストエリアの高さ**：現在 `height(500.dp)` で固定。画面サイズ対応の柔軟なレイアウトにしたい
- **Googleドキュメント連携フロー**：「コピーして戻ってくる」という複数ステップを、ガイドUI（ステップ表示など）で補助できると良い
- **空状態のデザイン**：Main画面でテキストが未入力の場合のEmpty Stateデザインが未整備
- **オンボーディング**：初回起動時の使い方案内が未実装
- **アクセシビリティ**：フォントサイズ切替は実装済み。さらなるアクセシビリティ対応（コントラスト・タッチターゲットサイズ等）の検討余地あり

## 6. 技術メモ（実装上の制約）

- **言語**：Kotlin / Jetpack Compose（Material3）
- **UIフレームワーク**：Compose の `LazyColumn`、`Scaffold`、`TopAppBar`（M3）
- **TTS**：Android標準の TextToSpeech サービスを `TtsService` としてバインド
- **データ永続化**：SharedPreferences（プロンプト・辞書・メインテキスト）
- **最小SDKバージョン**：確認中
